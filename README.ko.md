# ⎈ Helm

**AI 코딩 에이전트 CLI와 조화되는 가볍고 네이티브한 크로스플랫폼 대시보드-터미널.**

Claude Code, Codex, opencode를 Helm 터미널 안에서 실행하면 — 대시보드가 각
세션의 실시간 상태(진행, 작업, 도구, 플러그인, 토큰 컨텍스트)를 터미널 바로
옆에 보여줍니다. 에이전트 시대의 tmux/cmux.

<sub>`Tauri (Rust)` · `WebView2 / WKWebView / WebKitGTK` · `ConPTY / PTY` · `xterm.js + WebGL` · Electron 아님</sub>

## 왜

요즘 코딩 에이전트는 작업 목록, 도구 호출, 토큰 예산, 다단계 계획 같은 풍부하고
구조화된 출력을 내는 CLI입니다. 일반 터미널은 이 모든 걸 텍스트 스크롤백으로
납작하게 만듭니다.

Helm은 에이전트를 **진짜 터미널**에서 실행하면서(쉘에서와 똑같이 동작), 그와
동시에 그 구조를 **실시간 대시보드**로 재구성합니다 — 에이전트는 알 필요도
없습니다. 한 창, 여러 에이전트, 완전한 상황 인식.

에이전트가 지원하면 Helm이 **네이티브 후크**를 등록해 상태가 바뀌는 즉시
푸시합니다 — 폴링 없음. 지원하지 않으면 에이전트의 세션 로그를 파일시스템
이벤트에 깨어나 읽습니다. 어느 쪽이든 대시보드는 살아 있습니다.

## 무엇을 얻나

3분할 대시보드:

| 패널 | 내용 |
|---|---|
| **왼쪽** | 프로젝트별로 묶인 모든 세션, 상태 점과 실시간 활동 바. |
| **가운데** | 세션 탭(에이전트별 아이콘) + 실시간 터미널, 그리고 전사 기록을 메시지 카드로 다시 그리는 **터미널 ⇄ 대화** 토글, 모델 / 에이전트 / 추론 빠른 컨트롤이 있는 작성창. |
| **오른쪽** | 활성 세션의 실시간 상태 — 진행, 작업, 도구/플러그인, 토큰 컨텍스트 — 와 세션별 이벤트 타임라인, 실시간 시스템 CPU/MEM. |

## 기능

| 기능 | 설명 |
|---|---|
| **즉각 실시간 상태** | 후크를 지원하는 에이전트는 후크를 등록해 상태가 바뀌는 순간 푸시합니다 — 폴링 없음. 아니면 파일시스템 이벤트로 로그를 추적합니다. |
| **에이전트 조화** | 어떤 에이전트가 실행 중인지(입력 명령 + 터미널 제목) 감지해 세션에 라벨을 답니다 — 일반 쉘에서 직접 실행해도. |
| **작업 보드 (작업)** | "현재 활성" 개요: 실행 중인 세션마다 카드 하나에 상태, 실시간 활동, 초 단위 경과 타이머, 컨텍스트 사용 바 — 그리고 세션별 전체 작업 목록. |
| **대화 뷰** | 실시간 전사 기록을 읽기 전용으로 렌더 — 사용자/어시스턴트 카드, 접히는 추론, 도구 호출, 코드 블록. |
| **작성창 빠른 컨트롤** | 보내기 버튼 위의 모델 / 에이전트 / 추론 칩 — 실시간 목록에서 고르면 그 자리에서 전환. |
| **모바일 접속 (LAN)** | 같은 와이파이의 휴대폰으로 Helm을 열어 세션을 실시간 조작 — 터미널, 진행, 작업, 대화. [모바일](#모바일-접속) 참고. |
| **세션별 작업 폴더** | 새 세션이 시작될 디렉터리 선택 — 경로 붙여넣기 또는 네이티브 탐색. |
| **세션 자동 복원** | 세션(작업 폴더 + 에이전트)이 재시작·재부팅 후 돌아옵니다; Claude는 `--continue`로 이어집니다. |
| **클립보드** | 터미널에서 붙여넣기/복사 — 다른 앱에서 복사한 텍스트 포함(아래 단축키 참고). |
| **WebGL 렌더러** | 글리프를 GPU로 그려 빠른 에코, 컨텍스트 손실 시 DOM 렌더러로 깔끔히 폴백. |
| **설정** | 터미널 폰트, 기본 에이전트, 패널 표시, 복원 토글. |

## 에이전트 지원

Helm이 보여줄 수 있는 것은 각 에이전트가 디스크에 쓰거나 후크로 푸시하는 것에
달려 있습니다.

| 기능 | Claude Code | Codex | opencode |
|---|:---:|:---:|:---:|
| 감지 + 라벨링 | ✓ | ✓ | ✓ |
| 상태 / 활동 | ✓ | ✓ | ✓ |
| 작업(작업 목록) | ✓ | ✓ | ✓ |
| 도구 / 플러그인 | ✓ | ✓ | ✓ (MCP/플러그인) |
| 대화 뷰 | ✓ | ✓ | — ¹ |
| 토큰 컨텍스트 | ✓ | ✓ | — |
| 네이티브 후크로 즉각 | ✓ | ✓ ² | ✓ |
| 출처 | `~/.claude` 전사 + 후크 | `~/.codex` 롤아웃 로그 + 후크 | 디스크 로그 + 후크 플러그인 |

¹ opencode는 메시지 본문을 잠긴 데이터베이스에 보관하므로, Helm은 전체 대화
카드 대신 상태·활동·파일 작업·설정된 MCP/플러그인 목록을 보여줍니다.

² Codex의 작업과 도구는 롤아웃 로그에서 옵니다(파일시스템 이벤트로 깨어나 읽어
이미 거의 즉각적). 후크는 더 이른 상태/턴 신호를 더해 줍니다.

> **새 에이전트 추가**는 Helm의 정규화된 이벤트를 emit하는 워처(또는 후크
> 핸들러) 하나만 쓰면 됩니다 — UI는 어떤 에이전트든 일반적으로 렌더합니다.
> [작동 원리](#작동-원리) 참고.

## 모바일 접속

Helm은 **같은 와이파이**의 휴대폰에 전체 UI를 서빙하고, 모든 세션을 WebSocket으로
실시간 연결합니다 — 데스크톱이 쓰는 바로 그 이벤트 스트림.

**접속 방법:**

1. 상단 바의 **📱** 버튼을 누릅니다. 대화상자에 **URL**, LAN IP, 두 포트가
   나옵니다.
2. 휴대폰(같은 와이파이)에서 그 URL을 엽니다 — 또는 **복사**를 눌러 휴대폰
   브라우저에 붙여넣습니다.
3. 끝. 이제 휴대폰이 세션을 실시간으로 미러링·조작합니다: 터미널, 진행, 작업,
   대화.

**휴대폰 레이아웃**에서는 3패널이 하나의 전체 너비 컬럼으로 접힙니다
(터미널/대화가 중심). 상단 바의 두 버튼으로 사이드 패널을 서랍처럼 엽니다:

| 버튼 | 여는 것 |
|---|---|
| **☰** | 세션 목록(왼쪽 레일) |
| **📊** | 세션 상태 레일(오른쪽) |

어두워진 영역을 탭하거나, 세션을 고르거나, `Esc`를 누르면 서랍이 닫힙니다.

**페어링 & 포트:**

| 항목 | 값 |
|---|---|
| 페어링 | 실행마다 새 랜덤 토큰이 URL에 박힙니다; WebSocket은 그 외 모든 것을 거부합니다. |
| HTTP 포트 | `8787` — `HELM_HTTP_PORT`로 변경 |
| WebSocket 포트 | `8788` — `HELM_WS_PORT`로 변경 |
| 범위 | LAN 전용. 클라우드도 릴레이도 없음 — 휴대폰이 내 기기와 직접 통신. |

> 휴대폰이 페이지를 못 열면 방화벽이 포트를 막고 있을 가능성이 큽니다 — 개인
> 네트워크에서 Helm(또는 해당 포트)을 허용하세요. LAN 밖 접속을 위한 클라우드
> 릴레이는 [로드맵](#로드맵)에 있습니다.

## 키보드 & 클립보드

`Mod` = Windows/Linux는 `Ctrl`, macOS는 `Cmd`. 앱 단축키는 터미널 자체 바인딩
(readline 등)을 지키려고 일부러 단순 `Ctrl`+`<글자>`를 피합니다.

**세션**

| 키 | 동작 |
|---|---|
| `Ctrl` + `Tab` / `Ctrl` + `Shift` + `Tab` | 다음 / 이전 세션 |
| `Mod` + `1`–`8` | 세션 1–8로 이동 |
| `Mod` + `9` | 마지막 세션으로 이동 |
| `Mod` + `Shift` + `T` | 새 세션 |
| `Mod` + `Shift` + `W` | 활성 세션 닫기 |

**뷰 & 터미널**

| 키 | 동작 |
|---|---|
| `Mod` + `Shift` + `M` | 터미널 ⇄ 대화 전환 |
| `Mod` + `Shift` + `K` | 터미널 스크롤백 지우기 |
| `Mod` + `Shift` + `U` | 알림 — 패널 열기 / 최신 미확인으로 이동 |
| `Mod` + `,` | 설정 |
| `Mod` + `=` / `Mod` + `-` | 폰트 크기 키우기 / 줄이기 |
| `Mod` + `0` | 폰트 크기 초기화 |

**클립보드**

| 키 | 동작 |
|---|---|
| `Mod` + `V`, `Shift` + `Insert` | 터미널에 붙여넣기 |
| `Mod` + `Shift` + `C`, `Cmd` + `C` | 선택 영역 복사 |
| `Ctrl` + `C` | 그대로 통과시켜 에이전트 중단 |

붙여넣기는 bracketed-paste 모드를 존중하므로, 여러 줄 경로가 Claude Code 같은
TUI에서 한 줄씩 실행되지 않고 하나의 덩어리로 들어갑니다.

## 요구 사항

시스템 웹뷰가 있는 데스크톱 OS, 그리고 구동할 에이전트 CLI(`claude`, `codex`,
`opencode`)가 `PATH`에 있어야 합니다.

| OS | 웹뷰 | 추가 빌드 의존성 |
|---|---|---|
| **Windows** 10/11 | WebView2 (현재 Windows에 기본 설치) | — |
| **macOS** | WKWebView (내장) | — |
| **Linux** | WebKitGTK | `webkit2gtk-4.1`, `libgtk-3-dev`, `librsvg2-dev`, `build-essential` |

## 소스에서 빌드

```bash
cd src-tauri
cargo build --release        # Rust 툴체인 + OS 웹뷰 의존성 필요
```

프런트엔드는 빌드 시점에 바이너리에 임베드되므로, `ui/` 아래를 바꾸면 재빌드가
필요합니다. 산출물:

| OS | 바이너리 |
|---|---|
| Windows | `src-tauri\target\release\helm.exe` (또는 `run.cmd`) |
| macOS / Linux | `./src-tauri/target/release/helm` |

코드가 바뀔 때마다 Windows / macOS / Linux 매트릭스 빌드가 CI에서 돕니다.

## 설정

대부분의 옵션은 앱 내 **설정** 패널에 있습니다:

| 설정 | 기본값 |
|---|---|
| 터미널 폰트 | 시스템 모노스페이스 |
| 기본 에이전트 | `claude` |
| 패널 표시 | 모두 켜짐 |
| 세션 복원 | 켜짐 |

환경 변수:

| 변수 | 효과 |
|---|---|
| `HELM_HTTP_PORT` | 모바일 UI를 서빙하는 포트(기본 `8787`). |
| `HELM_WS_PORT` | 모바일 WebSocket 브리지가 듣는 포트(기본 `8788`). |
| `HELM_USAGE_PORT` | 설정 시, 해당 포트의 로컬 엔드포인트에서 계정 사용량 JSON을 읽어 사용량 패널을 표시. 미설정 → 패널 숨김, 요청 없음. |

## 작동 원리

모든 파싱은 Rust 백엔드에서 일어나며, **정규화된 이벤트 스트림**(PTY id 기준)을
emit합니다. UI는 이를 일반적으로 렌더합니다 — 그래서 같은 데이터가 모바일
클라이언트에도 그대로 흘러갑니다.

| 이벤트 | 페이로드 | 용도 |
|---|---|---|
| `pty-data:{id}` | `{ b64 }` | 터미널 출력 |
| `pty-exit:{id}` | — | 프로세스 종료 |
| `agent-progress:{id}` | `{ status, activity, todos[], tools[], context }` | 오른쪽 레일 상태 |
| `conv-msg:{id}` | `{ role, text, thinking?, tool_calls[], usage? }` | 대화 메시지 |
| `conv-tool:{id}` | `{ name, status, result }` | 도구 호출 결과 |
| `conv-reset:{id}` | — | 대화 초기화 |

`agent-progress`를 채우는 두 경로:

- **네이티브 후크** — Helm이 로컬호스트 리시버를 띄우고 각 에이전트에 후크를
  등록합니다(Claude의 프로젝트 후크, opencode의 자동 로드 플러그인, Codex의
  `hooks.json`). 라이프사이클 이벤트가 발생 즉시 푸시됩니다. 등록은 추가형이며
  에이전트의 기존 설정이나 신뢰 저장소를 절대 건드리지 않습니다.
- **로그 워처** — 후크가 담지 않는 것(토큰 컨텍스트, Codex 작업 목록, 대화
  본문)은 에이전트별 워처가 세션 로그를 읽고 파일시스템 이벤트로 깨어납니다.

모바일 브리지는 같은 스트림을 직접 구현한 HTTP + WebSocket 서버(추가 의존성
없음)로 미러링하고, 휴대폰의 명령을 백엔드로 다시 전달합니다.

## 프로젝트 구조

| 경로 | 내용 |
|---|---|
| `src-tauri/src/main.rs` | PTY 생성 + I/O, 시스템 통계, Tauri 명령, 백그라운드 폴러 |
| `src-tauri/src/agent_watch.rs` | 진행 + 대화를 이벤트로 정규화하는 에이전트별 로그 워처 |
| `src-tauri/src/hook_server.rs` | 로컬호스트 후크 리시버 + 에이전트별 후크 등록 |
| `src-tauri/src/mobile.rs` | 휴대폰 클라이언트용 LAN HTTP + WebSocket 브리지 |
| `ui/` | 프런트엔드(프레임워크 없는 바닐라 JS): `index.html`, `styles.css`, `app.js` |
| `ui/vendor/xterm` | 벤더링된 xterm.js + WebGL/fit/web-links 애드온 |

## 문제 해결

| 증상 | 해결 |
|---|---|
| 빈 / 흰 창 | 웹뷰 캐시 손상(보통 강제 종료 후). Windows에서 `%LOCALAPPDATA%\com.helm.app` 삭제 후 재실행. |
| 오른쪽 패널이 비어 있음 | 에이전트를 세션 **안에서** 실행하고, 세션 작업 폴더가 실제 에이전트 실행 위치인지 확인 — 로그 조회가 그 경로 기준입니다. |
| 휴대폰이 페이지를 못 엶 | 같은 와이파이인가? 개인 네트워크에서 포트를 허용했는가? [모바일](#모바일-접속) 참고. |
| 붙여넣기가 안 됨 | 터미널을 먼저 포커스한 뒤 `Ctrl`/`Cmd`+`V` 또는 `Shift`+`Insert`. |
| 사용량 패널 없음 | `HELM_USAGE_PORT`를 설정하지 않으면 정상(설정 참고). |

## 로드맵

- 더 많은 에이전트 — 각각 정규화된 이벤트를 emit하는 워처(또는 후크 핸들러) 하나.
- LAN 너머 모바일 — 네트워크 밖 접속을 위한 선택적 클라우드 릴레이.
- 더 풍부한 모바일 제스처와 오프라인 재연결.
