# ⎈ Helm

**AI 코딩 에이전트 CLI와 조화롭게 동작하는, 가볍고 네이티브한 크로스플랫폼 대시보드 터미널.**

Claude Code · Codex · opencode 를 Helm 터미널 안에서 실행하면 —
각 세션의 진행 상황·작업·도구·플러그인·토큰 컨텍스트를 터미널 옆에서 실시간으로
보여줍니다. 에이전트 시대를 위한 tmux/cmux.

<sub>`Tauri (Rust)` · `WebView2 / WKWebView / WebKitGTK` · `ConPTY / PTY` · `xterm.js + WebGL` · Electron 없음</sub>

## 왜 만들었나

요즘 코딩 에이전트는 풍부하고 구조화된 출력을 가진 CLI입니다 — 작업 목록,
도구 호출, 토큰 예산, 다단계 계획. 일반 터미널은 그걸 전부 텍스트 스크롤백으로
납작하게 만들어 버립니다.

**Helm은 에이전트를 실제 터미널에서 실행**(셸에서 돌리는 것과 100% 동일하게
동작)하면서, **동시에 에이전트 자신의 세션 로그를 읽어** 그 구조를 실시간
대시보드로 복원합니다. 에이전트는 그 사실을 몰라도 됩니다. 한 창에서, 여러
에이전트를, 완전히 파악하면서.

## 화면 구성

3-패널 대시보드:

| 패널 | 내용 |
|---|---|
| **왼쪽** | 모든 세션을 프로젝트별로 그룹화, 상태 점 + 실시간 활동 바. |
| **중앙** | 세션 탭(에이전트별 아이콘) + 실시간 터미널, **터미널 ⇄ 대화** 토글로 트랜스크립트를 메시지 카드로 렌더. |
| **오른쪽** | 활성 세션의 실시간 상태 — 진행·작업·도구/플러그인·토큰 컨텍스트 + 세션별 이벤트 타임라인 + 시스템 CPU/MEM. |

## 기능

| 기능 | 설명 |
|---|---|
| **에이전트 조화(harmonization)** | 어떤 에이전트가 도는지 감지(입력 명령 + 터미널 제목)해서 세션에 라벨링 — 일반 셸에서 직접 띄워도 잡아냅니다. |
| **실시간 에이전트 상태** | 각 에이전트 자신의 로그를 tail 해서 상태·작업·도구/플러그인 사용을 실시간 표시. |
| **대화 뷰** | 실시간 트랜스크립트를 읽기 전용으로 렌더 — USER/ASSISTANT 카드, 접히는 추론, 도구 호출, 코드 블록. |
| **작업 뷰** | 에이전트의 작업 목록을 상태(대기/진행중/완료)와 함께, 전체 세션 통합 표시. |
| **세션별 작업 폴더** | 새 세션이 시작될 디렉토리 지정 — 경로 붙여넣기 또는 네이티브 찾아보기. |
| **세션 자동 복원** | 세션(작업 폴더 + 에이전트)이 재시작/재부팅 후 자동 복원, Claude는 `--continue`로 이어서. |
| **클립보드** | 터미널에서 붙여넣기/복사 (아래 단축키 참고). |
| **WebGL 렌더러** | 글리프를 GPU로 그려 빠른 에코, 컨텍스트 손실 시 DOM 렌더러로 깔끔히 폴백. |
| **설정** | 터미널 글꼴, 기본 에이전트, 패널 표시, 복원 토글. |

## 에이전트 지원

Helm이 보여줄 수 있는 것은 각 에이전트가 디스크에 무엇을 쓰느냐에 달려 있습니다.

| 항목 | Claude Code | Codex | opencode |
|---|:---:|:---:|:---:|
| 감지 + 라벨링 | ✓ | ✓ | ✓ |
| 상태 / 활동 | ✓ | ✓ | ✓ |
| 작업 목록 | ✓ | — | — |
| 도구 / 플러그인 | ✓ | ✓ | ✓ (MCP/플러그인) |
| 대화 뷰 | ✓ | ✓ | — ¹ |
| 토큰 컨텍스트 | ✓ | — | — |
| 로그 출처 | `~/.claude` 트랜스크립트 | `~/.codex` rollout 로그 | 디스크 로그 + 설정 |

¹ opencode는 메시지 본문을 잠긴 DB에 보관해서, 전체 대화 카드 대신 상태·활동·
파일 작업 + 설정된 MCP/플러그인 목록을 표시합니다.

> **새 에이전트 추가**는 Helm의 정규화된 이벤트를 emit하는 워처 하나만 쓰면
> 됩니다 — UI는 어떤 에이전트든 일반적으로 렌더합니다. *동작 방식* 참고.

## 키보드 & 클립보드

| 키 | 동작 |
|---|---|
| `Ctrl`/`Cmd` + `V`, `Shift` + `Insert` | 터미널에 붙여넣기 |
| `Ctrl` + `Shift` + `C`, `Cmd` + `C` | 선택 영역 복사 |
| `Ctrl` + `C` | 그대로 통과 — 에이전트 인터럽트 |

붙여넣기는 bracketed-paste를 존중하므로, 여러 줄짜리 경로도 Claude Code 같은
TUI에서 줄단위 실행 없이 한 덩어리로 들어갑니다.

## 요구 사항

시스템 웹뷰가 있는 데스크톱 OS, 그리고 사용할 에이전트 CLI(`claude`, `codex`,
`opencode`)가 `PATH`에 있을 것.

| OS | 웹뷰 | 추가 빌드 의존성 |
|---|---|---|
| **Windows** 10/11 | WebView2 (최신 Windows엔 기본 설치) | — |
| **macOS** | WKWebView (내장) | — |
| **Linux** | WebKitGTK | `webkit2gtk-4.1`, `libgtk-3-dev`, `librsvg2-dev`, `build-essential` |

## 소스에서 빌드

```bash
cd src-tauri
cargo build --release        # 필요: Rust 툴체인 + OS별 웹뷰 의존성
```

프론트엔드는 빌드 시 바이너리에 내장되므로 `ui/` 변경 시 재빌드가 필요합니다.
결과물:

| OS | 바이너리 |
|---|---|
| Windows | `src-tauri\target\release\helm.exe` (또는 `run.cmd`) |
| macOS / Linux | `./src-tauri/target/release/helm` |

코드가 바뀔 때마다 CI에서 Windows / macOS / Linux 매트릭스 빌드가 돕니다.

## 설정

대부분의 옵션은 앱 내 **설정** 패널에 있습니다:

| 설정 | 기본값 |
|---|---|
| 터미널 글꼴 | 시스템 고정폭 |
| 기본 에이전트 | `claude` |
| 패널 표시 | 전체 표시 |
| 세션 복원 | 켜짐 |

선택적 환경 변수:

| 변수 | 효과 |
|---|---|
| `HELM_USAGE_PORT` | 설정 시, 해당 포트의 로컬 엔드포인트에서 계정 사용량 JSON을 읽어 사용량 패널을 표시. 미설정 시 패널 숨김 + 요청 안 함. |

## 동작 방식

모든 파싱은 Rust 백엔드에서 일어나고, **정규화된 이벤트 스트림**을 프론트엔드로
emit합니다(PTY id 기준). UI는 이를 일반적으로 렌더 — 그래서 추후 원격/웹
클라이언트도 동일한 데이터를 그대로 받을 수 있습니다.

| 이벤트 | 페이로드 | 용도 |
|---|---|---|
| `pty-data:{id}` | `{ b64 }` | 터미널 출력 |
| `pty-exit:{id}` | — | 프로세스 종료 |
| `agent-progress:{id}` | `{ status, activity, todos[], tools[], context }` | 오른쪽 패널 상태 |
| `conv-msg:{id}` | `{ role, text, thinking?, tool_calls[], usage? }` | 대화 메시지 |
| `conv-tool:{id}` | `{ name, status, result }` | 도구 호출 결과 |
| `conv-reset:{id}` | — | 대화 초기화 |

## 프로젝트 구조

| 경로 | 내용 |
|---|---|
| `src-tauri/src/main.rs` | PTY 생성 + 입출력, 시스템 통계, Tauri 커맨드, 백그라운드 폴러 |
| `src-tauri/src/agent_watch.rs` | 에이전트별 로그 워처 — 진행·대화를 정규화해 이벤트로 emit |
| `ui/` | 프론트엔드(바닐라 JS, 프레임워크 없음): `index.html`, `styles.css`, `app.js` |
| `ui/vendor/xterm` | 벤더링된 xterm.js + WebGL/fit/web-links 애드온 |

## 문제 해결

| 증상 | 해결 |
|---|---|
| 흰/빈 화면 | 웹뷰 캐시가 꼬인 상태(보통 강제 종료 후). Windows에선 `%LOCALAPPDATA%\com.helm.app` 삭제 후 재실행. |
| 오른쪽 패널이 비어 있음 | 에이전트를 세션 **안에서** 실행하고, 세션 작업 폴더가 에이전트가 실제로 도는 위치인지 확인 — 로그 조회가 그 경로 기준입니다. |
| 붙여넣기가 안 됨 | 터미널을 먼저 포커스하고 `Ctrl`/`Cmd`+`V` 또는 `Shift`+`Insert`. |
| 사용량 패널이 없음 | `HELM_USAGE_PORT` 미설정 시 정상(설정 참고). |

## 로드맵

- 더 많은 에이전트 — 각각 정규화 이벤트를 emit하는 워처 하나면 됨.
- 기존 이벤트 스트림을 받는 원격 / 웹 클라이언트.
- 모바일 컴패니언.
